DNS (Domain Name System)

DNS stands for Domain Name System. It acts like the phonebook of the Internet. When you type a website name into your browser, your computer does not understand the human-readable domain name. DNS translates domain names into IP addresses.

DNS Components

  • Domain Name
  • IP Address
  • DNS Resolver
  • Root DNS Server
  • TLD Server
  • Authoritative Name Server

DNS Resolution Process

  • User enters domain name in browser
  • Browser sends request to DNS resolver
  • Resolver queries Root DNS server
  • Root server directs to TLD server
  • TLD server directs to authoritative server
  • Authoritative server responds with IP address
  • Browser connects to the web server

HTTP stands for HyperText Transfer Protocol. It is used for communication between a web browser and a web server. HTTP is stateless, meaning each request is independent.

HTTPS

HTTPS is HTTP with encryption. It uses SSL/TLS to encrypt communication between the client and the server. HTTPS prevents eavesdropping and man-in-the-middle attacks.

How Websites Work

  • User types a URL in the browser
  • DNS resolves domain name to IP address
  • Browser sends HTTP request
  • Server processes the request
  • Server sends HTTP response
  • Browser renders HTML, CSS, and JavaScript
